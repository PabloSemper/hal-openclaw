
# Informe Diario — HAL
## Instrucciones
Cada vez que recibas este prompt, generá un informe diario para Pablo con las siguientes secciones:
### 1. Estado de infraestructura
- Ejecutá `tailscale status` y reportá dispositivos conectados/desconectados
- Ejecutá `systemctl --user status openclaw-gateway` y reportá estado del gateway
- Verificá uptime del NUC con `uptime`
### 2. Noticias de tecnología relevantes
Investigá (usando web search) novedades sobre:
- Inteligencia Artificial aplicada a negocios/estaciones de servicio
- MikroTik actualizaciones de seguridad
- Raspberry Pi nuevos proyectos o actualizaciones
- OpenClaw actualizaciones de versión, skills nuevos o cambios en la plataforma
- OpenClaw nuevas automatizaciones, integraciones o casos de uso documentados por la comunidad
- Modelos LLM recomendados para el gateway de OpenClaw: relación costo/rendimiento, nuevos providers soportados
- Comparativa de opciones pagas eficientes para el gateway (no solo gratuitas): precios, límites, calidad real
- OpenClaw OAuth y API keys: cambios en proveedores, nuevos planes o formas de autenticación convenientes
- Computer Vision / YOLO novedades
- Hailo AI aceleración novedades
- OpenVINO novedades o actualizaciones
### 3. Audio, transcripción y monitoreo
Investigá novedades sobre:
- AssemblyAI y Whisper: actualizaciones, mejoras de accuracy, soporte para español
- Modelos o herramientas superadoras de Whisper para transcripción en español
- Soluciones de grabación y monitoreo de audio en entornos industriales o retail
- Nuevas técnicas de identificación de locutor (speaker diarization) en español
- ElevenLabs nuevas features de voz o transcripción
### 4. Novedades de herramientas que usa Pablo
- Tailscale actualizaciones
### 5. Tareas pendientes
- Leé el archivo /home/pablo/clawd/TAREAS.md y listá las pendientes
### Formato del informe
Sé CONCISO. Máximo 2-3 líneas por tema. Si no hay novedad en algo, no lo incluyas. Siempre incluí la fecha y hora del informe.
